### 1. 问题：文章目录.md 链接显示问题分析

**用户问题描述**：
- 在 GitHub 仓库中打开 `文章目录.md` 时，部分文章链接显示为纯文本而非可点击链接
- 部分链接点击后显示 404 错误

**原因分析**：
1. **链接显示为纯文本**：文章标题中包含方括号 `[]`，破坏了 Markdown 链接语法 `[链接文本](URL)`，导致 GitHub 的 Markdown 解析器无法正确识别链接
2. **链接 404 错误**：脚本生成的 URL 与 Jekyll 实际生成的页面 URL 不匹配，文件名与 Front Matter 中的标题不一致

**处理建议**：
1. 修复文章标题中的特殊字符，将方括号替换为中文括号或进行转义
2. 统一文件名与文章标题，确保 URL 路径一致
3. 修改 `generate_toc.py` 脚本，对标题中的特殊字符进行转义处理

### 3. 修复记录：Markdown 链接语法问题二次修复

**问题反馈**：第1-6篇文章的链接显示为纯文本（如 `[标题](URL)`），而非可点击的链接格式

**根本原因**：文章标题中包含英文方括号 `[]`，破坏了 Markdown 链接语法。当标题为 `[第005期_xxx]` 时，生成的 Markdown 链接为 `[[第005期_xxx]](URL)`，导致 GitHub 的 Markdown 解析器无法正确识别。

**修复方案**：
- **修改文件**：`generate_toc.py`
- **修改内容**：将转义策略改为替换策略，将英文方括号替换为中文括号
  ```python
  # 原代码：转义方括号
  title = title.replace('[', '\\[').replace(']', '\\]')
  
  # 修改后：替换为中文括号
  title = title.replace('[', '（').replace(']', '）')
  ```
- **效果**：标题中的英文方括号 `[]` 被替换为中文括号 `（）`，Markdown 链接语法正常解析

**验证结果**：
- 生成的链接格式：`[第005期_国内可直连每月自动重置免费额度模型使用完整教程](/2026/04/21/第 005 期每月自动重置免费额度模型使用教程.html)`
- 链接可正常显示和点击

### 4. 分析：GitHub仓库运行机制问题

**问题反馈**：本地文件正常，但远程GitHub仓库中文章目录.md显示异常

**GitHub Markdown渲染机制分析**：

1. **GitHub仓库直接显示.md文件**：
   - 使用 **GFM (GitHub Flavored Markdown)** 渲染引擎
   - 当链接文本中包含方括号 `[]` 时，会破坏Markdown链接语法 `[链接文本](URL)`
   - 例如：标题 `[第005期_xxx]` 生成链接 `[[第005期_xxx]](URL)`，导致解析失败

2. **GitHub Pages + Jekyll页面生成机制**：
   - Jekyll 从 `_posts` 目录读取.md文件
   - 根据文件名生成页面URL：`YYYY-MM-DD-标题.md` → `/YYYY/MM/DD/标题.html`
   - URL中的"标题"部分直接来自文件名，而非Front Matter中的title字段

**问题根源定位**：

| 问题现象 | 根本原因 | 影响范围 |
|---------|---------|---------|
| 第1-5篇链接显示为纯文本 | 文章标题中包含英文方括号 `[]` | GitHub仓库直接查看.md文件 |
| 第6-7篇链接点击404 | 文件名与标题不一致，URL路径不匹配 | GitHub Pages网站访问 |

**修复验证**：
- ✅ 已修复 `generate_toc.py`：将标题中的方括号替换为中文括号
- ✅ 已统一文件名与标题格式
- ✅ 已重新生成文章目录.md
- ✅ 已推送至远程仓库

**关键结论**：GitHub仓库有两种不同的渲染机制：
1. **仓库直接查看**：使用GFM渲染.md文件内容
2. **GitHub Pages网站**：使用Jekyll构建静态HTML页面

两者的URL生成逻辑不同，需要同时确保：
- Markdown语法正确（避免特殊字符破坏链接）
- 文件名与标题一致（确保URL路径正确）
